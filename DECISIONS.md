## 2026-02-17 — Keep product interview-specific
**Decision:** Stay focused on behavioral interview coaching for now.

**Rationale:**
- Clearer wedge and messaging.
- Faster MVP delivery.
- Avoids scope creep into general communication coaching.

**Implication:**
- Non-interview scenarios are deferred.

---

## 2026-02-17 — One product for USA + India
**Decision:** Single product with two input modes:
- English mode (default)
- Code-mix mode (Sarvam-enabled)

**Rationale:**
- Keeps core UX unified.
- Global relevance with regional differentiation.

**Implication:**
- Core rubric and output stay the same; only input pipeline differs.

---

**Decision:** Maintain continuity through project docs and git checkpoints.

**Rationale:**
